# RAG 领域研究方法（通用）

本文件沉淀“RAG（Retrieval-Augmented Generation）领域”的通用研究方法与产出规范，用于后续对同领域不同项目/方案做**可对比、可复用**的研究。

> 关键原则（按用户要求）：
> - 文档中**不做**“用/不用”的最终决策；只给出建议、优势/劣势、适用条件与风险。
> - 需要固定表格：研究对象总览表、参数推荐表、评估指标表。
> - 若项目涉及 prompts，需单独文件汇总全部提示词。
> - 所有包含代码片段的文档：必须提供**源文件+行号**链接，可跳转回原始代码。

---

## 0. 研究输入（开始前先补全）

- 研究对象：repo
- 研究问题（domain problem）：要解决的子问题（例如 ingestion、chunking、retrieval、rerank、evaluation、NL2SQL…）
- 约束：
  - 语言/语料（中文优先？）
  - 成本上限（模型/推理/解析）
  - 既有基础设施与集成点（例如 DB/搜索/权限体系/文档源）

---

## 1. 目录与文件规范

### 1.1 项目归档路径

- 领域索引（本文件）：`solution/rag/README.md`
- 项目目录：`solution/rag/<project>/`
- 项目索引（必须）：`solution/rag/<project>/README.md`
- 专题文档：`solution/rag/<project>/<topic>.md`

### 1.2 每个项目（project）最小产出集（最低配）

> 你确认“可以”，因此后续 RAG 项目研究默认至少产出以下 5 篇（必要时可合并/拆分，但要覆盖这些维度）：

1. `llm-or-system-analysis.md`（或 `analysis.md`）：架构、模块与关键设计取舍
2. `document-pipeline.md`：加载/解析/清洗/去重/元数据/管道
3. `chunking.md`：分块策略、参数、中文优化、源码要点
4. `retrieval.md`：召回/融合/过滤/权限/重排（如有）
5. `evaluation.md`：指标体系、数据集、自动化评测流程、验收口径

### 1.3 Prompts 规范（如项目涉及提示词）

- 必须新增：`solution/rag/<project>/prompts.md`
- 该文件只做“提示词目录/清单/版本记录”，方便复用与回溯。

## 2. 研究流程（建议步骤）

1) **建立项目目录与 README 索引**：先搭骨架再填充内容
2) **快速通读与画组件图**：明确 ingestion → index → retrieval → generation → evaluation 的端到端数据流
3) **按固定维度拆解**（见 1.2）：保证不同项目之间可对比
4) **基于证据输出建议**：优势/劣势/适用场景/风险/迁移成本
5) **回填领域方法论**：更新本文件中的表格（参数/指标）和“领域共识/最佳实践”

---

## 3. 建议输出模板（每篇文档都尽量遵守）

每篇专题文档建议包含以下小节（可按内容删减，但结论优先）：

- TL;DR（3~7 条要点）
- 背景：该专题要解决的子问题是什么
- 设计/实现：关键流程与关键数据结构（配图/伪代码/接口）
- 优势
- 劣势/限制
- 适用场景（什么情况下值得用/学/借鉴）
- 风险与成本（性能/复杂度/维护/供应商锁定/中文效果）
- 参考与证据（链接）

---

## 4. 代码片段引用规范（强制）

只要文档里出现代码片段（无论来自哪个 repo），必须提供**源文件+行号**链接。

### 4.1 链接格式（GitHub 推荐）

- 使用 GitHub **permalink**（包含 commit hash），并带行号：
  - `https://github.com/<owner>/<repo>/blob/<commit>/<path>#L10-L42`

### 4.2 片段标注建议

在代码块上方或下方追加一行：

- `Source: <url>#Lx-Ly`

> 目的：你可以一键跳回原始上下文，不依赖二次搜索。

## 5. 固定表格（领域内）

> 注意：本文件**不维护** RAG 项目索引（项目索引统一在知识库根目录 `README.md` 维护）。

### 5.1 参数推荐表（可逐步完善）

| 场景 | 推荐 chunk_size | overlap | 说明 |
|---|---:|---:|---|
| 中文通用文档（起步配置） | 512 | 50 | 从简单开始，覆盖多数场景；后续以评估驱动调参 |

### 5.2 评估指标表（可逐步完善）

| 维度 | 指标/方法 | 说明 |
|---|---|---|
| 检索 | Recall@k / MRR / nDCG | 评估召回质量与排序 |
| 生成 | Faithfulness / Groundedness | 评估是否基于证据回答 |
| 端到端 | 任务成功率 / 人评 | 按业务任务定义验收 |
