## 📋 核心发现 Top 10

### 1. 双服务架构
- **API 服务**: FastAPI + Celery + Redis + PostgreSQL + Neo4j + Elasticsearch
- **沙箱服务**: 安全代码执行环境 (Python/Node.js)

### 2. 完整的记忆生命周期管理
- 9 种记忆类型 (工作/短期/长期/情景/外显/知觉等)
- 5 环节知识链路 (产生→存储→检索→使用→优化)
- ACT-R 认知模型驱动的遗忘引擎

### 3. LangGraph 图工作流
- 记忆写入图：1 个节点 (save_neo4j)
- 记忆读取图：8 个节点 + 条件路由
- 工作流引擎：26 种节点类型

### 4. RAG 双引擎
- 传统 RAG: Elasticsearch 向量检索
- GraphRAG: Neo4j 图检索 + 关系遍历
- 混合检索：加权融合 (向量 60% + 图 40%)

### 5. 后台任务系统
- 14 个 Celery 任务
- 4 个 Beat 定时任务 (30 秒/1 小时/24 小时/可配置)
- Redis 消息队列

### 6. 代码规模
- 730 个 Python 文件
- ~158,000 行代码
- 核心业务文件 555 个 (100% 覆盖)

### 7. 设计模式应用
- 工厂模式：MemoryClientFactory
- 策略模式：ForgettingStrategy
- 状态模式：LangGraph WriteState/ReadState
- 责任链模式：WorkflowExecutor
- 模板方法模式：ExtractionOrchestrator

### 8. 数据存储三剑客
- **PostgreSQL**: 结构化数据 (配置/用户/知识库)
- **Neo4j**: 图数据 (知识三元组/实体/关系)
- **Elasticsearch**: 向量数据 (文本嵌入/相似度搜索)

### 9. 第三方集成
- 飞书集成：文档同步/消息推送
- 语雀集成：知识库同步
- 内置工具：百度搜索/MinerU/TextIn OCR

### 10. 安全机制
- 双认证：JWT (管理端) + API Key (服务端)
- 沙箱隔离：chroot/setuid 安全执行
- 敏感数据过滤：统一异常处理
