## 🎯 研究目的

1. **架构设计学习**: 理解如何在极小代码规模下实现完整的 AI 助手功能
2. **技术选型参考**: 分析轻量级设计的决策和权衡
3. **可复用组件提取**: 识别可以应用到其他项目的模块化设计
4. **对比分析**: 与 markitdown 等项目进行架构对比

- 主代理引擎（core agent）
- 提供者系统（Providers）
- 频道系统（Channels）
- 技能系统（Skills）
- MCP 集成
- 会话管理
- 记忆系统

### 关键特性列表
- 超轻量级设计（~4000 行代码）
- 多平台支持（Discord, Slack, WhatsApp, Feishu, QQ, Email 等）
- MCP（Model Context Protocol）支持
- 多提供者支持（OpenAI, Anthropic, 智谱等）
- 子代理系统
- 心跳机制
- 技能热加载

### 对比项目推荐
